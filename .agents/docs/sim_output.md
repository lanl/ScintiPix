# Output Structures

This document describes the structure of simulation output files, including the datasets and their fields.

The output of a `ScintiPix` simulation run is organized into a directory structure under the run root, with various stages containing parquet files for different datasets, along with metadata files such as `manifest.json` and `config.yaml` to maintain reproducibility and traceability of the simulation runs.

```text
run_id/
  manifest.json         # lightweight machine-readable file index
  config.yaml           # human-readable simulation configuration
  run.nxs               # NeXus file with scientific metadata and provenance
  logs/
    sim.log
  macros/
    macro1.mac
    macro2.mac
  primaries/
    part-000.parquet
  sensorHits/
    output.tpx3
  timing/
    part-000.parquet
  secondaries/
    part-000.parquet
  simulatedPhotons/
    part-000.parquet
  transportedPhotons/
    part-000.parquet
  intensifiedPhotons/
    part-000.parquet
```

## Manifest Files

- `manifest.json`: A lightweight machine-readable file index that lists all the output files generated during a given simulation run. 

```json
{
  "ScintiPix_schema_version": "0.1.0",
  "run_id": "000123",
  "config_file": "config.yaml",
  "nexus_file": "run.nxs",
  "log_files": [
    "logs/sim.log"
  ],
  "macro_files": [
    "macros/macro1.mac",
    "macros/macro2.mac"
  ],
  "stages": {
    "primaries": "primaries/",
    "sensor_hits": "sensor_hits/",
    "photons": null,
    "secondaries": null,
    "timing": null,
    "simulated_photons": null,
    "transported_photons": null,
    "intensified_photons": null
  }
}
```

## Config Files
- `config.yaml`: A human-readable file that contains the configuration parameters used for the simulation run. This file is crucial for reproducibility and traceability, as it allows users to understand the settings and parameters that led to the generated outputs. These config files are too long to be included in this documentation, please refer to the various config files under `examples/yamlFiles/` for examples of the parameters and settings that can be configured for a `ScintiPix` simulation run.

## NeXus Files
- `run.nxs`: A NeXus file that contains scientific metadata and provenance information about the simulation run. This file is structured according to the NeXus data format, which is a common format for storing and sharing scientific data. The `run.nxs` file includes metadata about the simulation configuration, the generated datasets, and the relationships between different stages of the simulation. This file can be used for detailed analysis and traceability of the simulation results.

## Output Stages

All output files in each of the stages are parquet files with schema defined in the following sections.

`ScintiPix` will write simulation outputs at various stages of the simulation process. The main stages are:

- `primaries/`: Contains datasets related to the primary particles generated in the simulation.
- `sensorHits/`: Contains datasets related to the sensor response, including intensifier and Timepix hits.

`ScintiPix` has the option to write out optional stages if needed for troubleshooting or analysis purposes. These optional stages include:

- `timing/`: Contains datasets related to timing information if the simulation is configured to pulsed sources.
- `secondaries/`: Contains datasets related to the secondary particles produced from interactions of primaries.
- `simulatedPhotons/`: Contains datasets related to the optical photons generated and detected at the optical interface.
- `transportedPhotons/`: Contains datasets related to the optical transport of photons through the system.
- `intensifiedPhotons/`: Contains datasets related to the photons after they have passed through the intensifier stage.

Note:

- If the simulation is configured with a continuous source, the `timing/` stage will not be present in the output.

### primaries

The `primaries/` stage contains datasets related to the primary particles generated in the simulation. The main fields in this stage are:

- `gun_call_id`: An identifier for the generation of an event in GEANT4.
- `primary_track_id`: An identifier for the primary particle track in the GEANT4 simulation.
- `primary_species`: The particle species of the primary particle (e.g., neutron, gamma, proton, etc.).
- `primary_x_mm`: The x-coordinate of the primary particle's position in millimeters.
- `primary_y_mm`: The y-coordinate of the primary particle's position in millimeters.
- `primary_energy_MeV`: The energy of the primary particle in mega-electronvolts.
- `primary_interaction_time_ns`: The time at which the primary particle interacted in the scintillator in nanoseconds.
- `primary_created_secondary_count`: The number of secondary particles created by the primary particle.
- `primary_generated_optical_photon_count`: The number of optical photons generated by the primary particle.
- `primary_detected_optical_interface_photon_count`: The number of optical photons detected at the optical interface.

Notes:

- Source creation and pulse timing values are used internally to set the Geant4
  primary vertex time. They are not recorded in `/primaries`; the only primary
  timing field persisted there is `primary_interaction_time_ns`.

### secondaries

The `secondaries/` stage contains datasets related to the secondary particles produced from interactions of primaries. The main fields in this stage are:

- `gun_call_id`: An identifier for the generation of an event in GEANT4, linking secondaries to their corresponding primaries.
- `primary_track_id`: An identifier for the primary particle track in the GEANT4 simulation.
- `secondary_track_id`: An identifier for the secondary particle track in the GEANT4 simulation.
- `secondary_species`: The particle species of the secondary particle (e.g., neutron, gamma, proton, etc.).
- `secondary_origin_x_mm`: The x-coordinate of the secondary particle's origin in millimeters.
- `secondary_origin_y_mm`: The y-coordinate of the secondary particle's origin in millimeters.
- `secondary_origin_z_mm`: The z-coordinate of the secondary particle's origin in millimeters.
- `secondary_origin_energy_MeV`: The energy of the secondary particle at its origin in mega-electronvolts.
- `secondary_end_x_mm`: The x-coordinate of the secondary particle's end position in millimeters.
- `secondary_end_y_mm`: The y-coordinate of the secondary particle's end position in millimeters.
- `secondary_end_z_mm`: The z-coordinate of the secondary particle's end position in millimeters.

Notes:

- There is no timing information recorded for the secondaries. We are depending on the timing information between the primaries and the photons to infer timing information about the secondaries.
- If no usable end position was recorded for a secondary, the `secondary_end_*_mm` fields will be written as `NaN`.

### simulatedPhotons

The `simulatedPhotons/` stage contains datasets related to the optical photons generated in the scintillator and detected at the optical interface. The main fields in this stage are:

- `gun_call_id`: An identifier for the generation of an event in GEANT4.
- `primary_track_id`: An identifier for the primary particle track in the GEANT4 simulation, linking photons to their corresponding primaries.
- `secondary_track_id`: An identifier for the secondary particle track in the GEANT4 simulation, linking photons to their corresponding secondaries.
- `photon_track_id`: An identifier for the photon track in the GEANT4 simulation.
- `photon_creation_time_ns`: The time at which the photon was created in the scintillator in nanoseconds.
- `photon_origin_x_mm`: The x-coordinate of the photon's origin in millimeters.
- `photon_origin_y_mm`: The y-coordinate of the photon's origin in millimeters.
- `photon_origin_z_mm`: The z-coordinate of the photon's origin in millimeters.
- `photon_scint_exit_x_mm`: The x-coordinate of the photon's exit position from the scintillator in millimeters.
- `photon_scint_exit_y_mm`: The y-coordinate of the photon's exit position from the scintillator in millimeters.
- `photon_scint_exit_z_mm`: The z-coordinate of the photon's exit position from the scintillator in millimeters.
- `optical_interface_hit_x_mm`: The x-coordinate of the photon's hit position at the optical interface in millimeters.
- `optical_interface_hit_y_mm`: The y-coordinate of the photon's hit position at the optical interface in millimeters.
- `optical_interface_hit_time_ns`: The time at which the photon hit the optical interface in nanoseconds.
- `optical_interface_hit_dir_x`: The x-component of the photon's direction at the optical interface.
- `optical_interface_hit_dir_y`: The y-component of the photon's direction at the optical interface.
- `optical_interface_hit_dir_z`: The z-component of the photon's direction at the optical interface.
- `optical_interface_hit_pol_x`: The x-component of the photon's polarization at the optical interface.
- `optical_interface_hit_pol_y`: The y-component of the photon's polarization at the optical interface.
- `optical_interface_hit_pol_z`: The z-component of the photon's polarization at the optical interface.
- `optical_interface_hit_energy_eV`: The energy of the photon at the optical interface in electronvolts.
- `optical_interface_hit_wavelength_nm`: The wavelength of the photon at the optical interface in nanometers.

Notes:

- If no usable exit position was recorded for a photon, the `photon_scint_exit_*_mm` fields will be written as `NaN`.
- If no usable hit position was recorded for a photon at the optical interface, the `optical_interface_hit_*_mm` fields will be written as `NaN`, and the `optical_interface_hit_time_ns` field will also be written as `NaN`.
- The `optical_interface_hit_*` fields capture position, time, direction, polarization, energy, and wavelength at the optical-interface crossing.
- The timing information in `photon_creation_time_ns` and `optical_interface_hit_time_ns` share the same Geant4 event-local global-time basis, allowing for timing analysis of photon generation and detection.

### transportedPhotons

The `transportedPhotons/` stage contains datasets related to the optical transport of photons through a lens system using the ray-optics python library. This simulate the transportation of photons from the optical interface to the image intensifier plane. The main fields in this stage are:

- `gun_call_id`
- `primary_track_id`
- `secondary_track_id`
- `photon_track_id`
- `intensifier_hit_x_mm`
- `intensifier_hit_y_mm`
- `intensifier_hit_z_mm`
- `reached_intensifier`
- `in_bounds`

### intensifiedPhotons

The `intensifiedPhotons/` stage contains datasets related to the photons after they have passed through the intensifier stage. The main fields in this stage are:

- `gun_call_id`: An identifier for the generation of an event in GEANT4.
- `primary_track_id`: An identifier for the primary particle track in the GEANT4 simulation, linking intensified photons to their corresponding primaries.
- `secondary_track_id`: An identifier for the secondary particle track in the GEANT4 simulation, linking intensified photons to their corresponding secondaries.
- `photon_track_id`: An identifier for the photon track in the GEANT4 simulation.
- `output_x_mm`: The x-coordinate of the photon's output position on the intensifier in millimeters.
- `output_y_mm`: The y-coordinate of the photon's output position on the intensifier in millimeters.
- `output_time_ns`: The time at which the photon was output in nanoseconds.
- `output_decay_time_ns`: The decay time of the image intensifier for the photon in nanoseconds.
- `output_blur_x`: The x-component of the photon's output width.
- `output_blur_y`: The y-component of the photon's output width.

### sensorHits

The `sensorHits/` stage contains datasets related to the sensor. This simulation currently only supports single timepix3 sensor with a SPIDR readout, but should have timepix3 quad capabilties in the future. The main fields in this stage are:
- `x_pixel`: The x-coordinate of the pixel that was hit on the sensor.
- `y_pixel`: The y-coordinate of the pixel that was hit on the sensor.
- `time_of_arrival_clockticks`: The time at which the hit was recorded in clock ticks.
- `time_over_threshold_clockticks`: The time over threshold for the hit in clock ticks.

### Timing

The `timing/` stage contains datasets related to timing information if the simulation is configured to pulsed sources. This usually requires a `T_zero` signal that is used to measure the time-of-flight. This is recorded in clock ticks of the TPX3Cam SPIDR readout. The main fields in this stage are:
- `T_zero_time_clockticks`: The time of the T_zero signal in clock ticks.
